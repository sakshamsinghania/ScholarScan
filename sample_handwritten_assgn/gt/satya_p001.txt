1) Architecture of a Helm chart
→ A Helm chart is a package used to define install & manage kubernetes application, its architecture consists of following main components
• Chart.yaml - Contains metadata such as the chart name version & description.
• Values.yaml - stores default configuration values used in templates.
• Templates/ - Contains kubernetes resource definitions written as templates.
• Charts/ - Includes dependent charts (subcharts)
• Readme.md - Provides documentation about chart.
These components together make deployment reusable & easy to manage.
